The external review correctly noted that `L2` is the most visible missing rigor gate, but not the only remaining issue. The manuscript also needs stronger reporting around run accounting, selection logic, methods detail, and construct language.

Remediation:
- add explicit run-accounting memo
- add retained-selection reporting note
- add journal methods-reporting checklist
- revise manuscript language so `L2` is framed as the largest single gate, not the only gate

### 2. Selective-reporting risk around retained runs and curated pair sets

The current manuscript package uses disciplined internal retention, but the phrase `retained run` can sound like post hoc curation if the full run universe and the replacement logic are not surfaced explicitly.

Remediation:
- define what `retained` means operationally
- separate stable evidence-bearing runs from exploratory or superseded runs
- report the run universe at the branch level, not only the narrative level

### 3. Construct-validity risk around the word `teaching`

The current manipulation supplies reusable protocol artifacts in the taught condition. That is a meaningful effect, but it may be read as strong procedural scaffolding rather than durable learning or self-generated capability acquisition.

Remediation:
- tighten manuscript language around what is actually manipulated
- distinguish protocol injection, protocol reuse, and self-generated protocol learning
- reserve stronger `teaching` claims for the narrower sense already operationalized in the repo

Primary artifacts:
- future manuscript language revision note under S099
- future manuscript scope and claims update under S100

### 4. Methods transparency is below journal standard

The packet is internally organized, but a journal submission will need fuller reporting on run counts, settings, scoring logic, loop rules, failure handling, and reviewer procedures.

Remediation:
- establish a journal methods checklist
- use that checklist to drive manuscript revision and appendix construction
- make branch-specific reporting expectations explicit

Primary artifact:
- [methods-reporting-checklist-journal-v1.md](C:\Users\Dan\Desktop\Projects\Quantyra-Teaching\docs\evidence\methods-reporting-checklist-journal-v1.md)

### 5. The paper is too broad in its current form

The current manuscript tries to carry too many theses at once: protocol externalization, hosted cross-provider uplift, local resilience, and multi-model reviewer architecture. That breadth is useful internally but weakens an external journal narrative.

Remediation:
- narrow the main manuscript spine to hosted cross-provider within-model uplift
- keep local resilience as a bounded secondary section
- treat BP006 as a secondary architecture analysis unless later expanded substantially

- future scope narrowing decision under S100
- future venue memo under S101
