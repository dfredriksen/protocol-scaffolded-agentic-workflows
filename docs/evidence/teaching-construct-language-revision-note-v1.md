# Teaching Construct Language Revision Note v1

Date: 2026-03-15  
Scope: journal-facing construct discipline for the word `teaching`

## Purpose

This note tightens how the manuscript should talk about `teaching` so the claim matches the current manipulation and evidence. The goal is not to abandon the core concept. The goal is to stop the paper from sounding like it demonstrates durable autonomous learning when the main retained evidence is about performance uplift under externally supplied reusable protocol artifacts.

## Current Risk

The current manuscript can be read too broadly because:

- the taught condition directly supplies a reusable protocol artifact
- most main results are about within-model performance uplift under that supplied artifact
- self-generated protocol learning is planned elsewhere, not yet the main evidence spine

So a skeptical reader may conclude that the paper does not show `teaching` in the strong sense of durable internalized learning. They may conclude instead that it shows that explicit procedural scaffolding improves performance. That interpretation is partly correct and needs to be handled directly.

## Recommended Construct Position

For the current manuscript, `teaching` should mean:

- introducing a reusable protocol artifact that captures task-relevant procedure
- applying that artifact in later execution conditions
- measuring whether that explicit protocolization improves task performance, workflow discipline, or reviewer-mediated convergence

The manuscript should not imply, unless clearly qualified:

- durable internal learning
- autonomous skill acquisition from experience alone
- stable cross-session capability change without artifact reuse

## Recommended Main Framing

Safer external framing:

- `protocol-mediated performance uplift`
- `performance gains under reusable protocol artifacts`
- `workflow improvement under explicit protocolization`
- `teaching in the operational sense of externalized reusable procedure`

Less safe external framing:

- `the agent learned the skill`
- `the agent acquired the capability`
- `teaching causes durable improvement` without qualification

## Manuscript Guidance

### Abstract and Introduction

Use language that foregrounds:

- reusable protocol artifacts
- externally supplied procedures
- repeated within-model uplift under protocolized execution

Do not let the opening paragraphs imply that the main evidence is about internal learning dynamics.

### Methods

Define the taught condition explicitly as:

- task execution with a supplied reusable protocol artifact

Define the current self-generated learning gap explicitly as:

- not yet the main evidence spine of this manuscript

### Discussion

The discussion should say clearly:

- the current evidence supports protocol-mediated uplift
- this is consistent with one operational sense of teaching
- stronger claims about self-generated learning remain future work

## Safe Claim Ladder

### Safe

- externally supplied reusable protocol artifacts improve performance relative to weaker baseline prompting
- protocolization helps across multiple hosted providers
- local models also benefit in bounded ways, especially on concrete workflow tasks

### Conditional

- this operationalizes a constrained form of teaching because a reusable procedure is transferred into later execution

### Not Yet Safe

- the paper demonstrates durable autonomous learning from experience
- the paper demonstrates self-generated protocol learning as the primary result

## Journal Implication

The manuscript should present `teaching` as an operational term tied to reusable externalized procedure, not as a broad synonym for internal capability acquisition. That narrower framing is still interesting, publishable, and much easier to defend.
