# L2 Manuscript Integration Note V1

## Purpose
Define exactly how distinct-evaluator `L2` review results should be folded into the current manuscript packet once they arrive.

## Scope

This note applies to:
- hosted matrix `L2` review first
- any later local-branch `L2` follow-on if it is executed

The immediate priority remains the hosted matrix because that is the primary publication surface.

## Primary insertion points

### 1. Manuscript draft
Update:
- `docs/evidence/manuscript-draft-v1.md`

Sections most likely to change:
- Abstract
- Hosted Results
- Local and Resilience Results, only if `L2` changes cross-layer interpretation
- Threats to Validity
- Pending `L2` note
- Conclusion

### 2. Tables
Update:
- `docs/evidence/manuscript-tables-v1.md`

Expected changes:
- confirm or revise score interpretations in Table 1
- revise Table 4 validity language once `L2` exists

### 3. Claim matrix
Update:
- `docs/evidence/manuscript-claim-matrix-v1.md`

Expected changes:
- revise claim caveats
- revise next-gate language
- mark whether hosted claims can be strengthened

### 4. Figure captions
Update:
- `docs/evidence/figure-captions-draft-v1.md`

Expected changes:
- remove or soften draft-only caveats if the hosted matrix is confirmed strongly enough
- keep local caveats unless local `L2` exists

## Integration rules

1. Do not silently overwrite original operator-side scores.
   Record whether `L2`:
   - agrees
   - narrows uncertainty
   - changes interpretation

2. If `L2` changes a score materially, update:
   - the relevant table row
   - the relevant manuscript prose
   - the relevant claim-matrix entry

3. If `L2` broadly agrees with the hosted matrix, strengthen wording only one step:
   - from strong internal pilot evidence
   - toward stronger publication-facing pilot evidence
   Do not overcorrect into "final proof."

4. Keep hosted and local evidence separate.
   Hosted `L2` should not be used to imply local independent validation.

## Minimum `L2` reporting bundle needed for manuscript update

- reviewer identity
- independence level
- agreement or disagreement with stronger condition
- score-change summary if any
- per-run review memos or consolidated hosted review summary

## Recommended post-`L2` packet update order

1. update hosted synthesis memo if needed
2. update manuscript tables
3. update manuscript claim matrix
4. update manuscript draft
5. update figure captions
6. update manuscript packet

## Current recommendation

The manuscript package is now ready for `L2` integration work. Once the hosted `L2` review is complete, this note should be used as the update order rather than revising assets ad hoc.
