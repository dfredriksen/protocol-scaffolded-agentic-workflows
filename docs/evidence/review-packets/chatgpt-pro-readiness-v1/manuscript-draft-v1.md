# Teaching Agents Through Protocols: Evidence for Cross-Provider Uplift and Bounded Local Resilience

## Abstract
Agentic systems often fail through weak planning, poor approval-gate discipline, and incomplete handoff behavior. This repository investigates whether those failures can be reduced by treating "teaching" as protocol externalization rather than one-off prompt tuning: lessons are converted into reusable protocol artifacts, embedded into benchmark packets, and evaluated through retained baseline-versus-taught comparisons. The primary evidence layer is a hosted cross-provider matrix in which OpenAI, Google, and Anthropic models show repeated within-model uplift across planning and workflow benchmark families. The secondary evidence layer is a bounded local resilience branch in which open-weight models improve most credibly on concrete workflow artifact tasks rather than abstract repo-grounded planning. The strongest current publication-grade limitation is evaluator independence: the evidence remains predominantly `L0/L1` pending a distinct-evaluator `L2` review.

## 1. Introduction
The motivating problem for this program is not whether large models can occasionally complete a task. It is whether an agentic system can be taught to behave more reliably over time through explicit, reusable procedural structure. In this repo, "teaching" does not mean that a model improvised a better answer once. It means that a lesson was externalized into a protocol artifact that can be reused, benchmarked, and audited across future runs.

This framing matters because much of what appears to be agent improvement in practice is brittle prompt specialization. A model may look stronger on one task simply because a longer or more prescriptive prompt was supplied. The research question here is narrower and more operational: if lessons are turned into structured protocols with explicit constraints, validation checkpoints, fallback behavior, and retained outputs, do agent outcomes improve in a way that is repeatable across model types and benchmark families?

The repo now supports two distinct evidence layers. The primary layer is a hosted matrix across multiple providers, where the strongest current claim is repeated within-model uplift under the taught protocol condition. The secondary layer is a local open-weight branch, where the strongest current claim is bounded resilience: protocol teaching and review loops can preserve some useful improvement under weaker local-model and fallback-review conditions, but without approaching hosted-model quality or removing the need for stronger external review. In the current local branch, the strongest gains occur on concrete workflow artifact tasks rather than on abstract repo-grounded planning tasks.

## 2. Methodology
The program operationalizes teaching through a sequence of artifacts. First, a teaching episode records a task, the baseline failure mode, the lesson extracted, and the protocol steps that should be reused later. Second, the lesson is externalized into a protocol artifact rather than left as operator memory. Third, the protocol is embedded into benchmark packets that compare a weak-prompt baseline against a taught condition. Fourth, the resulting outputs are retained and scored using a consistent rubric.

The benchmark families in the current repo cover several different questions. `BP001` measures planning quality. `BP002` through `BP004` progressively strengthen workflow and artifact-handling demands, moving from structured workflow discipline toward higher-realism local artifact creation. `BP006` studies executor-reviewer architectures, including correction and takeover patterns. `BP007` and `BP007B` study quality-bar loops and reviewer strictness. `BP008` shifts the local branch toward concrete workflow artifact tasks after `BP007B` showed that stronger review alone did not rescue weak local planning quality.

All current evidence should be interpreted with one central caveat: most scoring is still operator-side. The repo includes rigor protocols and handoff assets for `L2` distinct-evaluator review, but those reviews have not yet become the dominant evidence tier. So the current manuscript posture is intentionally conservative: the repo supports strong internal claims and a structured publication path, but not yet fully publication-grade independence.

## 3. Hosted Results
The hosted matrix currently covers OpenAI `gpt-4.1`, Google `gemini-2.5-flash`, and Anthropic `claude-sonnet-4-6`. Across the retained `BP001` through `BP003` hosted run01 sets, every hosted provider showed within-model uplift in the taught condition relative to baseline. The most repeated pattern is not provider ranking but protocol effect: planning and workflow behavior improved when the protocol was explicit, especially when the packet required stronger state separation, approval-gate discipline, or handoff quality.

The strongest hosted planning signal appears in `BP001`, where weaker baselines leave more room for protocol lift. The strongest hosted workflow signal appears in the `BP002` through `BP004` progression, with the retained `BP003` hosted run01 set and the clarified `BP004` slice providing the clearest evidence that harder packets reduce ceiling effects and force cleaner artifact packaging, approval handling, and reviewer-safe handoff. The transfer variants, especially the retained `BP003` run02 set, further strengthen the workflow claim by showing the same protocol effect on adjacent workflow tasks rather than only one packet phrasing.

The hosted evidence therefore supports a publication-oriented main claim: protocol teaching improves within-model outcomes across multiple hosted providers on planning and workflow tasks. What it does not yet support is a fully independent publication-grade claim, because the decisive evaluator-independence step still depends on `L2` review.

## 4. Local and Resilience Results
The local branch now has three interpretable layers. The first is the open-weight `BP001` planning comparison. Across the retained five-model set, every open-weight model showed some within-model uplift there, but the strongest taught score plateaued at `6/10`, which is enough to support a limited low-parameter claim but not enough to justify immediate transfer or workflow expansion from planning alone.

The second layer is `BP007B`, the verifier-backed planning branch. That branch is methodologically useful because it solved the review-bar problem: false-positive stretch passes were eliminated once the reviewer had to check against the actual repo inventory. But the substantive outcome remained weak. Stronger review did not rescue local planning quality. The best retained verifier-backed point, `T07`, still only reached `pass-core`, and the weaker local executors in `T08` and `T09` failed the core bar outright.

The third layer is `BP008`, the workflow-task local branch. This is the strongest current local result surface. In `W01`, under hosted review, `llama3.2:3b` improved from `4/10` to `5/10` and reached `pass-core` on a concrete workflow artifact task. In `W03`, under an all-local fallback pair, the same executor improved from `2/10` to `4/10` and again reached `pass-core`. That all-local result is weaker than the hosted-review lane and should not be overstated, but it is still the first retained evidence that a local fallback path can preserve part of the workflow gain.

The main local conclusion is therefore more specific than a generic "open-weight models improve" claim. Local models improve most credibly when the task family is concrete and artifact-centered. In the retained local evidence, `BP007B` remains a negative-but-useful planning branch, while `BP008` is the strongest current workflow branch. Workflow tasks give weaker local models something they can actually repair: file presence, blocker labels, stop-condition logs, and handoff structure. Abstract repo-grounded planning remains a much harder local surface.

## 5. Mixture-of-Experts Results
`BP006` extends the teaching question beyond single-model taught conditions. The key result is that reviewer quality and executor strength interact. Strong executors paired with strong reviewers often benefit most from correction: the original model absorbs critique and produces the stronger final artifact. Weaker executors paired with stronger reviewers often benefit more from takeover: the second model becomes the final author because critique alone does not produce enough repair.

The repo now includes hosted-hosted, hybrid hosted-local, and fully local mixture-of-experts points. The pattern is useful for both science and deployment. It suggests that multi-model improvement is not a single architecture effect. Whether correction or takeover is superior depends on which model is strong enough to revise effectively and which model is strong enough to judge or rewrite reliably. The current BP006 result should therefore be read as a strong retained-pair pattern, not yet as a finished exhaustive law over the full planned pairing matrix.

## 6. Discussion
The current repo supports a coherent methodological claim: protocol teaching works best when it is treated as explicit procedural structure that can be reused, benchmarked, and audited. Hosted results show that this effect generalizes across multiple providers. Local results show that the effect can persist in weaker settings, but only under certain task families and with lower ceilings.

What the paper can claim now is relatively clear. First, hosted within-model uplift is repeated enough across `BP001` through `BP004` to support a real protocol-teaching effect rather than a one-off prompt artifact. Second, `BP006` shows that multi-model improvement is architecture-dependent: correction, takeover, and no-gain cases all appear, and their distribution depends on executor strength and reviewer quality. Third, the local branch supports a bounded resilience claim, especially once the benchmark surface shifts from abstract planning to concrete workflow artifacts.

What the paper should not claim yet is equally important. It should not claim that local models match hosted-model quality, that all review loops are equally effective, or that protocol teaching has already been validated under publication-grade independent scoring. The current evidence still contains operator-side scoring, some proxy-materialized workflow tasks, and local-runtime constraints that narrow the resilience interpretation.

Why the local branch still matters is strategic as well as scientific. It shows that the protocol methodology can preserve part of its value even when the system is degraded from hosted-primary operation to a fallback local-review mode. That is not a replacement-quality result, but it is a meaningful resilience result.

## 7. Threats to Validity
The strongest current threat is evaluator independence. Most scoring still comes from the program operator. The repo has already prepared `L2` review assets, but until that review is completed, the evidence should be treated as strong internal pilot evidence rather than final publication-grade proof.

There are additional methodological limits. The taught condition currently supplies the protocol directly, so the measured effect is partly a protocol-following effect rather than a fully separate controller effect. Some workflow results remain proxy-materialized rather than native live-tool execution. Local runs are also shaped by machine limits, including one-model-at-a-time storage discipline and slower local inference. None of these invalidate the current findings, but they do constrain how strongly the findings should be generalized.

## 8. Conclusion
The current strongest claim is that protocol teaching improves within-model outcomes across multiple hosted providers on planning and workflow tasks. The current strongest secondary claim is that bounded local resilience effects also exist, especially on concrete workflow artifact tasks where a protocol-plus-review loop can improve weaker local models and preserve some gain under an all-local fallback mode.

The repo is not yet at the point where local evidence should replace hosted evidence, nor at the point where evaluator independence can be assumed. The next publication-grade gate is still `L2` review. But the program now has something more valuable than scattered pilots: a coherent evidence structure, a disciplined publication posture, and a clear distinction between primary hosted claims and bounded local resilience claims.
