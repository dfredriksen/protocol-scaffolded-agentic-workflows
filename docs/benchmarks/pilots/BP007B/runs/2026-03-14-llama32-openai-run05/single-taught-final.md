# Single-Model Taught Final

## Objective
Advance the Quantyra Teaching research repo by planning the next benchmark phase for self-learning protocols and local-model evaluation.

## Ordered plan
1. Review the current benchmark framework and identify the next open items for self-learning and local evaluation.
2. Draft a benchmark packet for self-generated protocols so the model can attempt, reflect, and retry.
3. Add a run sheet for the next comparison cycle and record expected evidence outputs.
4. Update the repo checkpoint after the benchmark packet and run sheet are in place.

## Outputs
- benchmark packet documentation
- run-sheet documentation
- retained evidence notes
- checkpoint update

## Validation
- confirm the benchmark packet exists
- confirm the run sheet exists
- confirm the checkpoint is updated

## Risks and fallback
- if the benchmark packet is too broad, reduce it to one narrow pilot
- if the local model branch is unstable, keep the run as smoke evidence only
