# Next Implementation Phase Plan: Post-BP001 Freeze

## Objective
Advance Quantyra-Teaching by designing and benchmarking BP002, focusing on expanded reasoning tasks and improved story coverage, while maintaining the repository's markdown-based workflow.

## Constraints
- All work must be tracked and documented in markdown artifacts within the repository.
- No external systems, databases, or APIs.
- All benchmarks, stories, and decisions must follow established formats.
- Changes must not disrupt BP001 reproducibility.

## Required Artifacts
- `benchmark-packet-BP002.md`: Defines new or expanded benchmarks and evaluation criteria.
- `story-set-BP002.md`: Curated set of stories illustrating new reasoning tasks.
- `run-sheet-BP002.md`: Step-by-step instructions for running BP002 benchmarks.
- `decision-log-BP002.md`: Records key decisions, rationales, and trade-offs.
- `checkpoint-update-BP002.md`: Summarizes progress and interim results.

## Dependencies
- Finalized and frozen BP001 artifacts.
- Access to all markdown templates and formats used in BP001.
- Existing BP001 benchmark results for comparison.

## Ordered Execution Steps
1. Review BP001 decision logs and checkpoint updates to identify gaps and improvement areas.
2. Draft BP002 objectives and scope in `decision-log-BP002.md`.
3. Define new reasoning tasks and evaluation metrics in `benchmark-packet-BP002.md`.
4. Curate or author new stories in `story-set-BP002.md`.
5. Create `run-sheet-BP002.md` detailing execution steps for benchmarks and story evaluations.
6. Execute benchmarks as per run sheet and document results in `checkpoint-update-BP002.md`.
7. Update `decision-log-BP002.md` with findings, issues, and next-step recommendations.

## Validation Checkpoints
- All new artifacts conform to established markdown formats.
- BP002 benchmarks run reproducibly using only repository contents.
- Results and decisions are clearly documented and traceable.

## Fallback
If BP002 scope proves unmanageable or incompatible with BP001, revert to a minimal BP001.1 update focused on incremental benchmark clarifications.
