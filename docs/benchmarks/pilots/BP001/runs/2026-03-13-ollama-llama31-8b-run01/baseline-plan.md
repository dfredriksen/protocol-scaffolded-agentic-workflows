**Implementation Phase: Execution and Scoring of BP001 and BP002 Pilots**

**Task Setup**

1. Ensure the repository is up-to-date with the latest changes.
2. Identify the pilot teams for BP001 and BP002 and provide them with necessary instructions, pilot documents, and evaluation criteria.
3. Set up a shared drive or cloud storage for pilots to store their benchmark execution artifacts.

**Evaluator Workflow**

1. Assign evaluators to each pilot team, ensuring they understand their roles and responsibilities.
2. Evaluator workflow:
	* Review pilot plans and ensure they meet the evaluation criteria.
	* Conduct on-site or remote benchmark executions, as applicable.
	* Complete evaluator worksheets for each pilot, documenting observations, challenges, and lessons learned.
	* Submit worksheets and pilot plans to the repository for scoring.

**Retained Outputs**

1. **Run Folders**: Store each pilot's benchmark execution artifacts in a separate run folder, including:
	* Pilot plan and documents
	* Evaluator worksheets
	* Score summaries
2. **Evaluator Worksheets**: Store each evaluator's worksheets in a separate folder, including:
	* Evaluation criteria met/not met
	* Observations and lessons learned
3. **Score Summaries**: Compile score summaries for each pilot, using the retained outputs.

**Pilot Closeout Artifacts**

1. **Comparison Memo**: Prepare a comparison memo for each pilot, highlighting:
	* Key findings and conclusions
	* Recommendations for improvement
2. **Pilot Report**: Write a pilot report for each pilot, summarizing:
	* Pilot objectives and outcomes
	* Lessons learned and challenges
	* Recommendations for future improvements

**Execution and Scoring**

1. Execute the benchmark for each pilot, using the retained outputs as inputs.
2. Score each pilot based on the evaluation criteria, using the retained outputs and pilot plans.
3. Compile score summaries for each pilot, using the retained outputs.

**Artifacts**

1. Retain the following artifacts:
	* Run folders (pilot-specific)
	* Evaluator worksheets (evaluator-specific)
	* Score summaries (pilot-specific)
	* Comparison memo (pilot-specific)
	* Pilot report (pilot-specific)

This implementation phase ensures a structured approach to executing and scoring BP001 and BP002 pilots, leveraging the existing repository infrastructure and retained outputs.
