# L2 Hosted Execution Bundle V1

## Purpose
Provide one unambiguous entry point for executing the hosted distinct-evaluator `L2` review.

## Use this bundle when
- a distinct internal evaluator is available
- the hosted matrix is the review target
- the goal is to produce manuscript-update-ready `L2` outputs

## Reviewer entry sequence

1. open the reviewer `.docx` packet folder in the human `L2` bundle
2. read `H005-l2-reviewer-assignment-packet.docx`
3. read `H008-l2-reviewer-checklist-v1.docx`
4. read `l2-review-handoff-protocol.docx`
5. use the `review-artifacts` folder for the frozen hosted run evidence
6. score the frozen hosted runs only from retained artifacts
7. record outputs under the designated results location below
8. consult the completed `L2A` comparison material only after independent scoring, if helpful for comparison
